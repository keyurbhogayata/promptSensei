# Code Style and Conventions
- **TypeScript First:** Maintain strict typing.
- **Modularity:** Keep logic separated into functional modules within `src`.
- **Async/Await:** Use modern asynchronous patterns for file and git operations.
- **Testing:** Mirror `src` structure in the `tests` directory.
- **Git Commits:** Use imperative mood (e.g., `feat: add context optimizer`) and include scope if possible.
- **Documentation:** Maintain `README.md` and update `GEMINI.md` for architecture/convention changes.
