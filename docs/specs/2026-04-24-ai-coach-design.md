# AI Coach: Token & Effort Optimization Plugin

## 1. Overview
AI Coach is an agnostic development tool designed to analyze AI-assisted coding sessions post-completion. Its goal is to calculate wasted tokens, quantify the monetary cost of inefficient prompting, and provide actionable coaching advice to help developers write better prompts and adopt more systematic approaches.

## 2. Architecture & Integration
The system is built to integrate with various AI development environments (e.g., Claude Code, Gemini CLI, IDEs like VS Code with GitHub Copilot).

- **Core Engine:** A Node.js-based/Python-based CLI tool.
- **Interfaces:**
  - **CLI Wrapper:** Executable via terminal (e.g., `ai-coach review <path-to-chat-log>`).
  - **MCP Server:** Exposes tools via the Model Context Protocol (MCP) so compatible IDEs and CLIs can trigger reviews automatically at the end of a session.
- **Inputs:**
  - Chat history logs (JSON/Markdown formats depending on the host tool).
  - Git diffs (local file changes since the session started).

## 3. Data Flow & Waste Calculation

### 3.1 The Heuristics Calculator (Local)
Calculates exact waste metrics locally without incurring additional LLM costs.
- **Process:**
  1. Parses the chat log to map generated code blocks to specific turns.
  2. Compares generated code against the final Git working tree state.
  3. Identifies "churned" code (lines generated by AI that were heavily modified or deleted shortly after).
  4. Flags the tokens from the turns that produced the churned code as "wasted."
  5. Identifies "correction" patterns (e.g., user inputs containing "error", "no", "fix this").
- **Cost Calculation:** Multiplies wasted token counts by configurable model pricing (e.g., $3.00 / 1M input tokens, $15.00 / 1M output tokens).

### 3.2 The LLM Advisor (Remote/Local LLM)
Provides qualitative feedback on the identified wasted efforts.
- **Process:**
  1. Extracts only the context (user prompts and resulting errors/churn) of the flagged "wasted" turns.
  2. Sends this condensed context to a fast, cost-effective LLM (e.g., Gemini Flash or Claude 3 Haiku).
  3. Prompts the LLM to identify the missing context or poor phrasing that caused the waste.
  4. Generates a concise coaching tip.

## 4. User Experience (UX)

### 4.1 Trigger
When the host CLI or IDE detects the session is idle, completed, or a specific task is done, it presents a prompt:
`[AI Coach] Session ended. Would you like a token & prompt review? (Y/n)`

### 4.2 Output Report
If the user opts in, a brief, structured report is printed to the console or IDE output panel:

```text
📊 Session Review
- Total Tokens Used: 45,000
- Wasted Tokens: 12,000 (26%)
- Money Wasted: $0.04

💡 Coaching Advice:
In Turn 3, you asked to "update the database logic" but didn't specify the schema. This caused a hallucination and 3 wasted turns. 
Next time, try: "Update the database logic to match this schema: [paste schema]."
```

## 5. Future Expansion
- Integration with dashboards for team-wide token waste analytics.
- Real-time "prompt linting" before a message is sent.
